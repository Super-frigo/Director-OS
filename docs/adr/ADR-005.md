# ADR-005: Architecture Review System

Date: 2026-07-11

## Status

Accepted

## Context

Director OS 的目标不是自动生成视频。

它的目标是建立一个 AI-native filmmaking operating system。

此前架构已经定义：

* ADR-001: Project 保存完整创作意图。
* ADR-003: Engine 负责创作推理。
* ADR-004: Library 提供电影知识。

但是，一个完整的导演系统还缺少一个重要能力：

> 对创作过程和输出结果进行评估。

传统电影制作中，导演不断进行 Review：

* 这个镜头是否符合故事？
* 角色是否保持一致？
* 视觉风格是否统一？
* 情绪是否正确传达？
* 镜头语言是否服务叙事？

AI 视频时代，这个过程更加重要。

因为 AI 模型可以快速生成大量结果，但生成数量增加并不等于创作质量提高。

如果没有 Review System：

```text
Generate → Accept
```

系统只能执行。

因此 Director OS 必须拥有独立的 Architecture Review System。

## Decision

Director OS 引入 Review Layer。

Review Layer 的职责：

> 检查 Project、Engine Decision 和 Generation Result 是否符合系统定义的创作目标。

整体架构：

```text
Creative Intent

      ↓

Project

      ↓

Engine

      ↓

Compiler

      ↓

Generation

      ↓

Review System

      ↓

Feedback Loop
```

Review System 是系统的质量控制层。

## Responsibilities

## 1. Validate Creative Consistency

Review System 检查：

* Character consistency
* Story consistency
* Visual consistency
* Emotional consistency

例如：

Project 定义：

```yaml
character:
  personality:
    calm
    intelligent
```

生成结果：

角色表现：

* exaggerated emotion
* aggressive behavior

Review System 应发现偏差。

---

## 2. Validate Production Decisions

Review System 检查 Engine 输出：

例如：

Engine:

```yaml
camera:
  extreme_close_up
```

Review:

```text
Potential issue:

Scene emotion is loneliness.
Extreme close-up may reduce emotional distance.

Recommendation:
Consider wider composition.
```

Review 不直接创作。

它提供反馈。

---

## 3. Validate Model Output

Review System 可以分析：

* Generated video
* Images
* Metadata
* Motion consistency

检查：

* Visual match
* Character identity
* Camera execution
* Style adherence

---

## 4. Create Feedback Loop

Review System 的核心价值：

不是评分。

而是改进。

完整循环：

```text
Project

 ↓

Engine

 ↓

Compiler

 ↓

Generation

 ↓

Review

 ↓

Improvement

 ↓

New Generation
```

## Architectural Rules

## Rule 1: Review Does Not Modify Creative Truth

Project 仍然是唯一真相源。

Review 可以提出：

* warnings
* suggestions
* evaluations

但不能直接修改 Project。

关系：

```text
Project:
"This is the creative intention"

Review:
"Does output match the intention?"
```

---

## Rule 2: Review Is Independent From Generation

Review 不能依赖：

* specific AI model
* compiler implementation
* generation provider

否则：

Review 会成为模型测试工具。

而不是创作审查系统。

---

## Rule 3: Review Should Be Multi-Dimensional

电影质量不能由单一指标决定。

Review 应支持多个维度：

```text
Review Dimensions:

├── Narrative
├── Character
├── Visual Style
├── Cinematography
├── Emotion
├── Consistency
└── Technical Quality
```

---

## Rule 4: Review Results Become Knowledge Input

Review 产生的数据可以进入：

```text
Review

 ↓

Knowledge Improvement

 ↓

Library

 ↓

Better Engine Decisions
```

但必须经过整理。

Review 不是自动修改知识库。

## Alternatives Considered

## Alternative 1: Let Human Review Everything

Rejected.

原因：

AI 视频生成速度远超人工处理能力。

系统需要辅助筛选和分析。

---

## Alternative 2: Let Engine Review Its Own Output

Rejected.

原因：

同一个系统负责生成和评价容易产生偏差。

需要独立审查边界。

---

## Alternative 3: Use Simple Quality Score

Rejected.

原因：

电影质量不是单一数字。

例如：

```text
Visual Score: 95

Story Score: 40
```

仍然可能是一部失败作品。

## Consequences

## Positive

* 建立完整创作闭环
* 支持持续优化
* 提高 AI 输出可靠性
* 接近真实导演工作方式

## Negative

* 增加系统复杂度
* 需要定义评价体系
* 需要视觉和文本分析能力

## Neutral

Review System 会成为 Director OS 的反馈神经。

## Implementation Notes

推荐目录：

```text
director-os/

├── projects/

├── engines/

├── libraries/

├── compilers/

├── reviews/

│   ├── narrative/
│   ├── visual/
│   ├── character/
│   ├── consistency/
│   └── technical/

└── schemas/
```

Review 输出格式：

```yaml
review:
  target:
    scene_id: scene_001

  dimensions:
    visual:
      score: 8

    emotion:
      score: 6

  issues:
    - character inconsistency

  suggestions:
    - adjust lighting
```

## Summary

Generation alone does not create a Director OS.

A director does not only create.

A director evaluates, corrects, and improves.

Review System gives Director OS the ability to understand the difference between:

```text
Generated Output
```

and

```text
Intended Film
```

The complete architecture becomes:

```text
Project
   ↓
Engine
   ↓
Compiler
   ↓
AI Model
   ↓
Review
   ↺
Improvement
```

Review System is the feedback mechanism that allows Director OS to continuously improve.
