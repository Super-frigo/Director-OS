# ADR-003: Engine Layer Introduction

Date: 2026-07-11

## Status

Accepted

## Context

Director OS 的目标不是简单连接用户与 AI 视频模型。

它需要成为一个 AI-native filmmaking system。

根据此前架构决策：

* ADR-000 定义了 Director OS 存在的原因：连接人类创意与 AI 生产能力。
* ADR-001 定义了 Project 是整个系统唯一的创作真相源。
* ADR-002 定义了 Compiler 必须保持隔离，不能让模型需求污染创作结构。

然而，在 Project 与 Compiler 之间仍然存在一个关键问题：

> 谁负责理解项目内容，并将导演意图转化为可执行的生产决策？

如果没有这一层，系统会退化为：

```
Project
   |
   |
Compiler
   |
AI Model
```

Compiler 将被迫承担：

* 理解故事
* 分析角色
* 决定镜头
* 推断视觉风格
* 生成生产策略

最终导致 Compiler 从转换层变成智能决策层。

这违反 ADR-002 的设计原则。

因此，Director OS 需要引入独立的 Engine Layer。

## Decision

Director OS 引入 Engine Layer，作为：

> 理解 Project、执行导演逻辑、产生生产决策的智能中间层。

新的架构：

```
Human Creative Intent

         ↓

Project

         ↓

Engine Layer

         ↓

Compiler

         ↓

AI Model
```

Engine 是 Director OS 的核心推理层。

## Responsibilities

Engine Layer 负责：

### 1. Project Understanding

Engine 读取 Project，并理解：

* Story structure
* Character relationships
* Scene context
* Shot requirements
* Visual language
* Emotional objectives

Engine 不修改 Project 的核心事实。

它基于 Project 进行分析。

---

### 2. Creative Reasoning

Engine 将抽象创作意图转换为生产决策。

例如：

Project:
```yaml
scene:
  emotion: loneliness
  location: abandoned_station
  character: traveler
```

Engine:
```
Recommended direction:

- Wide composition
- Slow camera movement
- Low contrast lighting
- Large negative space
```

这些属于导演执行策略，而不是 Project 本身。

---

### 3. Production Planning

Engine 可以生成：

* Shot breakdown
* Camera recommendations
* Visual consistency rules
* Generation strategy
* Model capability mapping

但最终输出必须保持平台无关。

---

### 4. Knowledge Resolution

Engine 通过 Knowledge Resolution 层请求知识（ADR-008）。

```
Project

  ↓

Engine

  ↓

Knowledge Request

  ↓

Resolution Pipeline
     ├── Local Rules
     └── Cached LLM

  ↓

Resolved Knowledge

  ↓

Creative Decision
```

Engine 是知识的消费者，不是知识的存储者。它通过 Resolution 接口请求知识，但不关心知识来自本地规则还是 LLM 缓存——这些是 Resolution 层的职责。

当 Engine 需要的知识在本地规则中不存在时，Resolution 层可以从 LLM 获取（并缓存结果）。Engine 自身不直接调用 LLM。

## Architectural Rules

## Rule 1: Engine Does Not Replace Project

Project 是事实。

Engine 是解释。

关系：

```
Project = What exists

Engine = How to execute
```

Engine 不应该成为新的数据源。

---

## Rule 2: Engine Does Not Replace Compiler

Engine 决定：

```
What should be created
```

Compiler 决定：

```
How a specific model receives it
```

例如：

Engine:
```
Create cinematic slow push-in shot
with emotional isolation.
```

Compiler:
```
Seedance:
camera_motion=push_in

Veo:
slow cinematic dolly movement
```

职责必须分离。

---

## Rule 3: Engine Output Must Remain Model Independent

Engine 输出不能包含：

* Seedance parameters
* Veo prompt syntax
* Kling specific controls
* Runway API fields

错误：
```yaml
camera:
  seedance_motion: slow_push
```

正确：
```yaml
camera:
  movement: slow_push
```

---

## Rule 4: Engine Should Be Composable

未来 Engine 不应该是单一巨大模块。

推荐：
```
engines/

├── story_engine
├── character_engine
├── visual_engine
├── shot_engine
├── consistency_engine
└── production_engine
```

不同 Engine 负责不同推理领域。

---

## Rule 5: Engine Requests Knowledge, Does Not Store It

Engine 通过 Knowledge Resolution 接口请求知识（ADR-008）。

Engine 不直接访问 LLM，不管理缓存，不决定知识来源。这些是 Resolution 层的职责。

Engine 只关心：给定当前 Project 上下文，我需要什么知识来做出更好的创意决策？

## Alternatives Considered

## Alternative 1: Put Intelligence Inside Compiler

Rejected.

原因：

* Compiler 会绑定模型
* 创意逻辑无法复用
* 多模型支持困难

---

## Alternative 2: Put Intelligence Inside Project

Rejected.

原因：

Project 应保存事实，而不是推理结果。

否则：
```
Project = Database + AI Brain
```

导致职责混乱。

---

## Alternative 3: Let Human Handle All Reasoning

Rejected.

原因：

AI filmmaking 的规模会超过人工管理能力。

复杂项目需要系统化辅助。

## Consequences

## Positive

* 保持 Project 长期稳定
* Compiler 保持简单
* 支持复杂 AI filmmaking workflow
* 为未来 Agent 系统提供基础
* 可以逐步增加智能能力

## Negative

* 增加系统复杂度
* 需要设计 Engine API
* 需要定义 Engine 间协作方式

## Neutral

Engine 将成为 Director OS 中最接近"导演大脑"的部分。

## Implementation Notes

推荐目录：
```
director-os/

├── projects/

├── engines/

│   ├── story/
│   ├── visual/
│   ├── character/
│   ├── shot/
│   └── production/

├── knowledge/          ← Knowledge Resolution 层

│   ├── providers/
│   │   ├── local_rules/
│   │   └── llm_cache/
│   ├── resolver.py
│   └── providers.py

├── compilers/

└── schemas/
```

Engine API 应类似于：

```
Engine(Project)
         ↓
Production Intent
```

而不是：

```
Engine(Prompt)
         ↓
Video
```

## Summary

Engine Layer exists because AI filmmaking requires more than data storage and model conversion.

Project preserves creative truth.

Engine interprets and expands creative intent.

Knowledge Resolution provides the filmmaking knowledge Engine needs (ADR-008).

Compiler translates production decisions into model-specific instructions.

Together they form the core architecture of Director OS:

```
Project
    ↓
Engine
    ↓
Knowledge Resolution     ← ADR-008
    ↓
Compiler
    ↓
AI Model
```

Engine is the intelligence layer between human creativity and machine generation.
