ADR-000: Why Director OS Exists
Status

Accepted

Context

The emergence of AI video generation fundamentally changes filmmaking.

Traditional filmmaking depends on:

Camera teams
Lighting departments
VFX pipelines
Editing systems
Production management

AI video models replace many execution steps.

However, current AI video workflows introduce a new problem:

The bottleneck is no longer execution.
The bottleneck becomes:

Maintaining creative consistency
Managing complex scenes
Preserving characters
Controlling visual language
Scaling from idea to complete film

Existing solutions focus on prompt generation.
But prompts are not the source of filmmaking.
A prompt describes a moment.
A film requires a system.

Decision
Director OS is created as a creative operating system for AI-native filmmaking.

Its responsibility is to manage the complete transformation:

Creative Intent
         →
Director OS
         →
AI Production Models
         →
Film

Director OS does not replace the director.
It extends the director's ability to:

Think
Organize
Iterate
Preserve intent
Coordinate AI tools

The full set of architecture principles that govern this system are
defined in [ARCHITECTURE_PRINCIPLES.md](../architecture/ARCHITECTURE_PRINCIPLES.md).

Consequences
Positive

Enables long-term AI filmmaking workflows
Separates creativity from model vendors
Allows multiple AI models to cooperate
Creates reusable creative assets

Negative

Requires more structured thinking
Requires building new abstractions
Initial development complexity increases

Summary
Director OS exists because AI filmmaking needs more than generation.
It needs a system that allows directors to create, manage, and evolve
films in an environment where execution is increasingly performed by machines.

Director OS is not a prompt tool.
It is the operating system between human imagination and AI production.
