# ADR-004: Library Driven Knowledge System

Date: 2026-07-11

## Status

Accepted

## Context

根据之前的架构决策：

* ADR-001 定义 Project 是唯一创作真相源。
* ADR-003 引入 Engine Layer 作为导演逻辑与创作推理层。

然而，一个关键问题仍然存在：

> Engine 如何获得专业电影知识？

一个没有知识来源的 Engine，只能依赖：

* 固定规则
* 硬编码逻辑
* 简单模板

这会导致：

```text
Engine = Automation Script
```

而不是：

```text
Engine = Creative Intelligence
```

AI-native filmmaking 需要的不只是生成能力，而是大量结构化专业知识：

* 电影摄影语言
* 镜头设计原则
* 叙事结构
* 角色塑造方法
* 视觉风格规律
* 类型电影经验
* AI 模型能力知识

这些知识不能全部写死在 Engine 中。

因此，Director OS 需要一个独立的 Library Driven Knowledge System。

## Decision

Director OS 引入 Library Layer。

Library Layer 负责：

> 存储、组织、检索和组合电影创作知识，为 Engine 提供专业能力。

整体架构：

```text
Project

   ↓

Engine

   ↓

Library System

   ↓

Creative Knowledge

   ↓

Production Decision
```

Library 是 Engine 的知识来源。

Engine 是 Library 的使用者。

## Definition

Library 不是：

* Prompt 集合
* 图片素材库
* 模型参数列表
* 静态模板仓库

Library 是：

> 可被 Engine 理解和组合的结构化创作知识单元。

例如：

一个 Cinematography Library Item：

```yaml
name: lonely_character_wide_shot

category: cinematography

knowledge:
  purpose: express isolation

visual_rules:
  framing: wide
  composition: negative_space
  movement: slow_push

emotion:
  loneliness: high
```

Engine 可以根据 Project 情境调用该知识。

## Responsibilities

## 1. Store Domain Knowledge

Library 保存长期稳定的电影知识：

例如：

### Cinematography Library

包含：

* Camera movement
* Lens language
* Composition patterns
* Lighting approaches

### Story Library

包含：

* Narrative structures
* Character arcs
* Conflict patterns

### Style Library

包含：

* Visual aesthetics
* Color philosophy
* Artistic references

### Character Library

包含：

* Character archetypes
* Personality structures
* Relationship patterns

---

## 2. Provide Contextual Knowledge Retrieval

Library 不应该简单返回搜索结果。

它应该支持：

```text
Project Context

        ↓

Knowledge Retrieval

        ↓

Relevant Creative Rules
```

例如：

Project:

```yaml
emotion: tension
location: dark_room
character: detective
```

Library 返回：

```yaml
knowledge:
  lighting:
    - high_contrast
    - hard_shadow

  camera:
    - close_frame
    - restricted_movement
```

供 Engine 推理。

---

## 3. Enable Knowledge Composition

电影创作不是单一规则匹配。

多个知识单元需要组合：

```text
Character Knowledge

        +

Visual Style Knowledge

        +

Camera Knowledge

        ↓

New Creative Decision
```

Library 必须支持组合，而不是简单读取。

## Architectural Rules

## Rule 1: Library Is Separate From Project

Project 不保存完整知识。

错误：

```yaml
project:
  camera_rules:
    use_wide_angle_for_loneliness
```

正确：

```yaml
project:
  emotion:
    loneliness
```

然后：

```text
Project
   ↓
Engine
   ↓
Library
   ↓
Camera Decision
```

原因：

Project 描述这个电影。

Library 描述电影知识。

---

## Rule 2: Library Is Not Prompt Storage

Prompt 是 Compiler 的输出形式。

Library 保存：

```text
Creative Meaning
```

而不是：

```text
Model Instruction
```

错误：

```yaml
prompt:
 "cinematic shot, 35mm lens..."
```

正确：

```yaml
concept:
  intimate_character_observation

principles:
  shallow_depth
  restrained_camera
```

---

## Rule 3: Engine Owns Reasoning, Library Owns Knowledge

职责：

```text
Library:
"What is known"

Engine:
"What should be done"
```

Library 不做决策。

Engine 不存储知识。

---

## Rule 4: Libraries Should Be Extensible

未来可以增加：

```text
libraries/

├── cinematography/
├── storytelling/
├── character/
├── style/
├── genre/
├── production/
└── model_capability/
```

新知识不应该修改 Engine 核心代码。

## Alternatives Considered

## Alternative 1: Hardcode Knowledge Inside Engine

Rejected.

原因：

* 难以维护
* 无法扩展
* Engine 变得臃肿

---

## Alternative 2: Store Everything Inside Project

Rejected.

原因：

* 项目文件巨大
* 知识无法复用
* 创作资产无法沉淀

---

## Alternative 3: Use External General Knowledge Database

Rejected.

原因：

通用数据库无法表达电影创作结构。

Director OS 需要的是：

```text
Creative Knowledge Graph
```

而不是：

```text
Document Search
```

## Consequences

## Positive

* Engine 获得专业能力
* 创作知识可以持续积累
* 不同项目共享经验
* 系统可以不断进化

## Negative

* 需要设计知识格式
* 需要维护 Library Quality
* 知识建模复杂度增加

## Neutral

Library 将成为 Director OS 的长期资产。

随着时间增长：

```text
More Projects

        ↓

More Knowledge

        ↓

Better Engines

        ↓

Better Films
```

## Implementation Notes

推荐目录：

```text
director-os/

├── projects/

├── engines/

├── libraries/

│   ├── cinematography/
│   ├── storytelling/
│   ├── characters/
│   ├── styles/
│   ├── genres/
│   └── model_capabilities/

├── compilers/

└── schemas/
```

Library Item 应保持：

* Human readable
* Machine interpretable
* Version controlled

推荐格式：

```yaml
id:
category:
concept:
principles:
constraints:
examples:
```

## Summary

Director OS cannot become intelligent by adding more prompts.

It becomes intelligent by accumulating structured creative knowledge.

Project stores the film.

Engine reasons about the film.

Library provides the knowledge.

Together they create a continuously improving AI filmmaking system.

```text
Project
   ↓
Engine
   ↓
Library
   ↓
Compiler
   ↓
AI Model
```

Library is the memory of Director OS.
