# 动画帧率速查表 (Frame Rate Quick Reference)

> 人工参考文档，不参与 Engine 自动检索（文件名以 REF_ 开头会被 LibraryLoader 跳过）。

| 风格 | 常用帧率 | 说明 |
|------|---------|------|
| 2D 手绘（Ones） | 24fps on-ones | 每帧一张原画，动作最流畅，成本最高 |
| 2D 手绘（Twos） | 24fps on-twos (实际12张/秒) | 经典动画节奏，略带停顿感，主流做法 |
| 三渲二 / Cel-shaded 3D | 24fps on-twos 渲染 | 用3D流程渲染出2D帧率的观感 |
| 定格动画 | 24fps on-twos，偶尔 on-ones 用于关键动作 | 逐帧实拍，on-ones 成本极高 |
| Motion Graphics | 24-30fps，关键帧驱动而非逐帧 | 依赖缓动曲线而非帧率本身 |
| 写实3D动画（长片/剧集） | 24fps，部分高帧率项目用 48-60fps | 高帧率会削弱"电影感"，需谨慎使用 |
| 游戏过场/实时渲染 | 30-60fps | 与互动性挂钩，非叙事性考量 |

## 选择建议

- 想要"经典动画感"、略带跳跃的节奏 → on-twos
- 想要极致流畅、贴近真人动作 → on-ones（成本会显著上升）
- 数据/UI 动画不适用传统"帧率美学"讨论，应优先考虑缓动曲线（easing）设计
